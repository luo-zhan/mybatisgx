# 0.1.0/0.1.0-sb4（2026-05-23）

## 🚀 性能优化

- 优化批量参数预处理及批量执行流程，提升批量操作性能。
- 值处理器静态绑定到字段元数据，避免运行期间重复从容器查找实例。
- 路由执行器由有状态实现调整为无状态实现，减少对象状态维护开销。
- 新增 `MethodCommandType`，避免执行阶段重复从 `SqlCommandType` 推导方法类型。
- 优化字段填充机制：
    - 基于 `LambdaAccessorFactory` 实现字段访问；
    - 字段元数据增加访问链缓存；
    - 提升复杂对象属性填充性能。
- 简单类型与复合类型统一采用 Lambda 访问方式。
- 优化批量关联查询加载流程：
    - 复合主键统一采用 `_` 分隔；
    - 使用 Lambda 动态调用提升数据装载性能。
- 实体参数元对象改为懒加载模式，减少无效对象创建。
- 字段信息静态化，避免值生成阶段重复创建字段描述对象。
- 静态分析方法是否需要值处理，不需要处理的方法直接跳过值处理流程。
- 对无需动态解析的方法提前编译动态标签，避免执行期间重复解析。

## ✨ 功能增强

- 查询操作支持无查询条件方法。
    - 删除和更新操作仍然要求必须存在查询条件。
- 元数据增加：
    - 是否存在查询条件标识；
    - 分页参数标识；
    - 在解析阶段完成预计算，减少执行期开销。
- 增加字段 Getter / Setter 方法合法性校验。
- 增加外键作为查询条件场景支持（实体参数及单参数形式）。

## 🏗️ 架构调整

- 优化路由执行器结构，提升可维护性。
- 方法元数据与实体元数据统一迁移至 `MybatisgxConfiguration` 管理。
- 字段处理逻辑从实体信息中拆分至独立字段信息处理器。
- 模板字段访问路径统一使用字段元数据中的访问链信息。
- 值处理执行时机由 `newStatementHandler` 调整至 `newParameterHandler`。

## ⚠️ 破坏性改动

- `FieldInfo` 从 `com.mybatisgx.executor` 包迁移至 `com.mybatisgx.model` 包，使元数据模型职责更加清晰。
- `ValueProcessor` 接口调整：
    - `phases()` 更名为 `commandTypes()`；
    - 返回类型由阶段枚举调整为 `MethodCommandType`。

## ✅ 测试增强

- 增加分页场景单元测试：
    - 有查询实体；
    - 无查询条件。
- 增加外键查询场景单元测试：
    - 实体参数；
    - 单参数。

## 🐛 问题修复

- 修复修改、删除、查询条件无法支持复合类型的问题。
- 修复动态条件无法访问任意层级属性的问题。
- 修复新增模板中逻辑删除 ID 未自动生成的问题。
